❌
HIGH → 修正済
cidExtractor: パストラバーサルシーケンスをそのまま通過 (両リポジトリ)
src/lib/cidExtractor.js / lib/cidExtractor.js (修正前)
../evil.dcv を入力すると { cid: '../evil', label: '../EVIL' } を返していました。 このcidがスクレイピングURLに埋め込まれると https://www.dmm.co.jp/.../cid=../evil/ という不正なURLが生成されます。 Electronアプリ側ではURLバリデーション修正で防げますが、ソース元での検証が不足していました。
修正内容: filename.replace(/.*[/\\]/, '') でパス区切り文字以降のファイル名のみを使用するよう修正。

renamer.js / PowerShellスクリプト生成テスト

  • ✅ タイトルに " が含まれる場合 → sanitize() で除去済み
  • ✅ タイトルに / が含まれる場合 → sanitize() で除去済み
  • ✅ 女優名が空配列の場合 → [LABEL] タイトル.dcv 形式で正常生成
  • ✅ sanitize() で \ / : * ? " < > | を正しく除去
⚠️
MEDIUM → 修正済
PowerShellスクリプト生成: r.filename のサニタイズ不足
pages/index.jsx:96-101 (Webアプリ)
renamer.js の sanitize() は newName に適用されますが、元のファイル名 r.filename はサニタイズ前のユーザー入力をそのまま使用。 テキストエリアに手動で " を含むファイル名を入力した場合、生成されるPowerShellスクリプトのコマンドが破損します。
修正内容: PowerShellスクリプト生成時に " を除去するサニタイズを追加。
